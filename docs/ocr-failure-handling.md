# OCR 失败与可见停机

用户原话：

> “还是频繁报告闪退问题，虽然有部分人可以通过依赖检测，但是等待10秒后还是会闪退。我觉得你的依赖检测不过关。另外就是加强一下防崩溃，OCR崩溃不要带到主程序，主程序先停下然后显示信息就行。比如校验失败，OCR崩溃（然后写上崩溃信息）”

目标行为：

- 启动阶段的 OCR 自检不能只检查文件和语言包，还要真实启动 OCR 后端并跑一次小图识别请求。
- 运行中 OCR 后端崩溃、初始化失败、请求超时、无返回、UI 缓存保险校验失败时，自动流程必须停止。
- 顶层用户可见程序不能静默退出。它要在控制台显示“已停止”的类别、错误摘要、日志位置，并等待用户确认关闭。
- 衔接启动的隐藏子程序不能等待用户输入。它只写日志并退出非 0，由总控读取同目录 `debug/last-error.txt` 给出摘要。
- `debug/last-error.txt` 保留完整异常；必要时额外写 `debug/fatal-stop-last.txt`，方便用户复制文本反馈。
