# 当前完整流程规格

## 默认规则

- 打开程序后先选择运行模式：`1` 正常模式，`2` 调试模式，`3` 重设设置。
- 如果选择 `3`，删除 `config/user-settings.json`，重新询问玩家“我的车辆页面中能够看到完整的几行、几列车辆”，重新框选完整可见车辆格子的整体区域，然后回到运行模式菜单。框选页面参考 `assets/vehicle-grid-page.png`。
- 如果选择 `1` 或 `2`，再检查 `config/user-settings.json`。
- 如果该文件不存在，询问玩家“我的车辆页面中能够看到完整的几行、几列车辆”。只统计右侧完整车辆格子；左侧车辆详情栏、最左/最右边缘露出的半个格子都不要算。
- 如果该文件来自旧版本，缺少 `dpi_aware_coordinates` 或 `calibration_mode=full_grid_v1` 标记，也会作废并重新框选，避免 Windows DPI 缩放或单格框选误差导致截图区域偏移。
- 支持任意分辨率，前提是首次运行或重设设置时按当前画面正确框选完整车辆格子区域。玩家更改分辨率、窗口缩放或 UI 比例后，需要重新选择 `3` 重设设置。
- 建议玩家把画质调到最低或较低，减少菜单动画和加载耗时，避免性能差的电脑超过脚本预留等待时间。
- 蓝图代码 `175 882 167` 必须已经加入“我的收藏”，并且处于“我的收藏”的第一位。程序进入收藏蓝图列表后会直接选择第一项，参考 `assets/favorite-blueprint-first.png`。
- 确认开始后的 `10` 秒等待结束时，游戏必须处于“大世界”可操作状态，参考 `assets/overworld-start.png`。
- 车库里有其它车辆或干扰车辆不会影响流程；脚本依赖首次框选和 OCR 动态识别，只要求前提条件满足。
- 然后弹出框选层，要求玩家用鼠标框选“所有完整可见车辆格子的整体区域”：从最左上完整格子的左上角拖到最右下完整格子的右下角。不要框左侧车辆详情栏、顶部制造商标签栏、底部按键提示，也不要框边缘不完整的半格。
- 弹出框选层前需要按 `Enter` 确认；确认后控制台隐藏，再开始框选。
- 框选层会同步显示起点、当前拖动点和蓝色矩形。
- 保存 `visible_rows`、`visible_columns`，并用整体区域按行列数切分出单格 `left/top/width/height` 后继续。
- 如果该文件存在，后续直接使用保存的行列和格子尺寸。
- 每次启动会询问玩家当前技术点总数；该值只存在于本次进程内，不写入文件，直接回车默认 `999`。
- 每处理完一辆车，技术点总数扣 `32`。
- 每轮主循环开头先检查剩余技术点；如果剩余技术点小于 `32`，脚本自动停止。
- 普通模式启动后等待 `10` 秒，然后进入主循环。
- 调试模式跳过开局 `10` 秒等待，直接进入主循环。
- 调试模式会保存每次 OCR 的截图链路到 `debug/screenshots/`，包含原始截图、实际送入 OCR 的缩放图、OCR 原始返回和元信息。
- 下一次调试模式启动时，会先清空 `debug/screenshots/`。
- OCR 后端默认使用 PaddleOCR PP-OCRv5；普通模式截图只在内存中传给 OCR，不保留调试截图。
- 未特别说明时，鼠标点击、键盘按键按下时间都是 `0.1` 秒。
- 未特别说明时，连续相同操作间隔都是 `0.1` 秒。
- 立即急停快捷键：`Space+C`，直接退出当前脚本。
- 安全急停快捷键：`Space+V`。如果当前在主程序且没有子程序运行，直接退出；如果子程序正在运行，主程序只向子程序发送安全退出请求，等子程序完成当前轮并复位后退出主程序。

## 主循环

### 1. 进入制造商列表并选择斯巴鲁

保留主循环最前面的 `Enter`。

1. 按 `Enter`。
2. 等待 `1` 秒。
3. 按 `Backspace`，打开制造商页面。
4. 等待 `0.5` 秒，让制造商页面稳定。
5. 鼠标移动到屏幕中心，保证滚轮作用在制造商列表上。
6. 鼠标滚轮向下滚动到底。
7. 整屏 OCR 找 `斯巴鲁` 并点击，进入斯巴鲁制造商车辆列表。

这一段已经替换掉原来的方向键录制/回放和更早的图寻制造商逻辑：

```text
Enter
寻找“图一标识”
鼠标移动到对应 UI 按键
循环点击并寻找“斯巴鲁”
找到后再次点击进入斯巴鲁车辆列表
OCR 找斯巴鲁
```

除此以外，后续流程保持原定逻辑。

### 2. 找到第一处 IMPREZA 22B-STI

1. 开始寻找 `IMPREZA 22B-STI`。
2. 每轮先 OCR 识别 `IMPREZA 22B-STI`。
3. 如果没找到，鼠标滚轮向下一格。
4. 找到后停止滚动。
5. 如果找到多个，按“优先左，其次上”选择：

```text
rank = x * 1000 + y
```

注意：找到的这一轮不再滚动。

## 虚拟格子算法

`GridGeometry` 不再通过 OCR 车名推断格子大小，也不再为了暴露三列指定车辆而额外滚动。

几何信息来自首次设置时玩家框选的最左上角完整格子：

```text
cellLeft = grid_cell_left + col * grid_cell_width
cellTop  = grid_cell_top  + row * grid_cell_height
hitRect[row][col] = cellLeft/cellTop/grid_cell_width/grid_cell_height
```

`visible_rows` 和 `visible_columns` 也来自首次设置。99% 情况下行数是 `3`，但允许玩家输入其它行数以兼容特殊分辨率比例。

保存后几何立即锁定。后续只更新格子内容，不重新建表、不重新估算格子尺寸。

## 后续循环识别

建好 `GridGeometry` 后，后续不再重新建表。

运行期虚拟列表使用三种状态：

```text
1 = 其他车辆，或未识别为目标车辆
2 = 目标车辆 IMPREZA 22B-STI，但没有待处理的 全新
3 = 目标车辆 IMPREZA 22B-STI 且有 全新，代表待点技术点
```

独立启动时不会读取旧虚拟列表文件；全自动每轮进入自动点技能点时也不会读取旧表内部车辆状态，而是重新 OCR 扫表并覆盖 `state/virtual-vehicle-list.json`。同一轮后续的删车衔接和删车后的找状态 `5` 会读取该文件，复用本轮已经建立并更新过的车辆状态。

如果虚拟列表里已经有 `3`，下一轮进入斯巴鲁列表后优先直接滚动到能看到这个 `3` 的位置，然后按方向键选中处理，不做 OCR。

固定操作完成后，本次处理的格子从 `3` 变回 `2`。

如果固定按键序列完成后已经能确认完成边界，例如最后已知列是 `331`，脚本会直接停止，不额外按 `Esc`，因为固定序列本身已经复位。

如果进入下一轮后没有可点车辆，OCR / 虚拟列表确认已经没有状态 `3`，这时脚本没有执行点技能点固定序列，需要额外按 `Esc`，等待 `1` 秒回到标准位，然后自动停止。

自动停止规则：

1. 虚拟列表里不存在 `3`。
2. 找到最后一个 `2`。
3. 车辆顺序按地平线列表规则计算：先在同一列从上到下，满三行后进入下一列。
4. 最后一个 `2` 的下一个格子必须已经存在，且状态是 `1`。
5. 满足以上条件时，脚本在当前轮固定操作完成后停止。
6. 如果最后一个 `2` 的下一个格子不存在，不停止，因为新列可能还有新车。

只有虚拟列表里没有 `3` 时，才跳过已知非 `3` 区间；跳到会露出新列的位置后，必须 OCR 更新新进入视野的格子。

状态 `0` 是边界信号，但不是“可见 `0` 直接退出”。用户原话：能见到 `0` 代表后续不会再有 `2/3/4`，把当前列表内的搞完就不用继续。

落实到虚拟表判断：

1. 当前可见范围内如果第一个 `0` 之前还有状态 `3` / `4` / `5`，必须先处理这些目标。
2. 第一个 `0` 之前如果还有未知格子，必须先 OCR 当前可见目标段。
3. 第一个 `0` 之后的格子不再要求 OCR，也不能作为后续目标。
4. 第一个 `0` 之前已经没有可处理目标时，才停止向后滚动或进入默认第一格逻辑。

每次滚动后只更新 `CellOccupancy`，也就是二维 bool 表：

```text
isTarget[row][col] = true / false
```

### 1. 更新目标车占用表

每次找 `全新` 前：

1. OCR 当前屏幕时走统一封装，一次性识别 `IMPREZA 22B-STI`、`全新`、`斯巴鲁`、`600`、`900`。
2. 目标车名先精确匹配，失败时允许小范围英文模糊匹配；中文字段先精确匹配，失败时允许小范围中文模糊匹配。
3. 同时 OCR 识别当前车辆列表内的 `斯巴鲁`。
4. 只有格子内识别到 `斯巴鲁` 的非目标车才记为状态 `1`；没有 `斯巴鲁` 的格子记为状态 `0`，不参与后续连续区间跳过。
5. 如果没有识别到可处理的 `斯巴鲁`，或者只识别到全局列表第一个占位格是 `斯巴鲁`，说明已经滚出斯巴鲁区域或没有剩余车辆，自动点技术点脚本会先按 `Esc` 复位到标准位，再停止当前子程序。
6. 把能映射进虚拟格子的目标车格子标成 `true`。
7. 其它格子默认 `false`。

这样即使车辆数量变化、排序变化，也不会拿旧内容硬算。

### 2. 找全新

1. OCR 识别所有 `全新`，先精确匹配，失败时允许小范围中文模糊匹配。
2. 每个 `全新` 坐标映射到虚拟格子：

```text
全新坐标 -> 命中的 hitRect -> row/col
```

3. 只有这个格子的：

```text
isTarget[row][col] == true
```

才认为这个 `全新` 属于目标车。

4. 如果该格子 `isTarget[row][col] == false`，忽略。
5. 如果本屏幕没有有效 `全新`，且没有触发斯巴鲁边界停止，就鼠标滚轮向下一格，然后重新更新 `CellOccupancy` 并继续找。

### 3. 选择有效目标

如果有多个有效 `全新`，按格子坐标选最左最上的：

```text
rank = col * 1000 + row
```

## 方向键移动到目标格

默认选中位置是当前屏幕从左到右第一列、第一行。

虚拟表格是 `visible_rows` 行、`visible_columns` 列：

```text
targetCol = 目标列，从 0 开始
targetRow = 目标行，从 0 开始，最大值是 visible_rows - 1
```

方向键移动：

```text
Right x targetCol
Down  x targetRow
Enter
等待 1 秒
```

每次方向键按下 `0.1` 秒，连续方向键间隔 `0.1` 秒。`Enter` / `Esc` 这类会打开或关闭菜单的操作后面至少等待 `1` 秒。

目标必须在当前屏幕可见范围内；如果不在，继续滚动寻找，不执行方向键移动。

## 当前关键假设

- 主循环开头的 `Enter` 保留。
- 制造商定位段使用 `Enter -> 等待 1 秒 -> Backspace -> 等待 0.5 秒让制造商页面稳定 -> 鼠标移到屏幕中心 -> 滚动到底 -> OCR 点击斯巴鲁`。
- 虚拟格子几何来自首次鼠标框选的最左上完整格子，不再用 OCR 推断。
- 虚拟列表第一个格子是永久占位车，永远不会被点技能点或删除。
- OCR 自动写表不会反复写已知格子；删除车辆导致的表格变化由删除模块修改表格，不属于 OCR 模块。
- 用户修正原话：“跟第一列没关系，总之就是OCR自动写表不会反复写已知格子。”
- “最左侧保留 1 列已知格子”只影响滚动策略和未知内容扩展，不影响已经写入虚拟列表的状态 `3`。如果已知状态 `3` 当前可见，即使位于这列，也必须直接选中处理；只有全局第一个占位格永远不处理。
- 同一车辆格子同时识别到 `全新` 和 `600` / `900` 时，自动点技术点阶段按状态 `3` 处理；状态 `4` 只能来自没有有效 `全新` 的目标车。
- 激进滚动的目标是让当前屏幕显示尽可能多的未知列，同时最左侧保留 1 列已知格子。
- 如果从当前 offset 开始的连续已知区长度为 `run`，实际滚动 `run - 1` 列；滚动后最左列是最后一列已知列，未知列从第 2 列开始进入 OCR 范围。
- 叠加层会用不同颜色细框标出本轮 OCR 识别到的符合条件字段，便于检查车名和 `全新` 是否识别正确。
- 格子绝对像素坐标在同一电脑上固定。
- 车辆内容会变化，所以每轮需要更新 `CellOccupancy`。
- OCR 失败时会把本轮识别文本和坐标写入 `debug/ocr-last-*.txt`，用于调整识别规则。
