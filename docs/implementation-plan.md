# 实际落地方案

## 当前进度

C# + PaddleOCR PP-OCRv5 OCR-only 版本已落地。

## 已定流程

按 `flow-spec.md` 实现。

核心点：

1. 启动后先选择运行模式：`1` 正常模式，`2` 调试模式，`3` 重设设置。
   - 如果选择 `3`，删除 `config/user-settings.json`，重新创建设置后回到运行模式菜单。
2. 选择 `1` 或 `2` 后检查 `config/user-settings.json`。
   - 如果文件不存在，询问“我的车辆页面能看到几行、几列完整车辆格子”。
   - 然后让玩家鼠标框选最左上角完整车辆格子，并保存 `visible_rows`、`visible_columns`、左上格子 `left/top/width/height`。
   - 如果文件存在，直接读取保存的行列和格子尺寸。
3. 正常模式启动等待 `10` 秒；调试模式跳过开局等待。
4. 每次启动询问当前技术点总数；这是本次进程内临时值，默认 `999`，不保存。
5. 主循环开头先检查剩余技术点；小于 `32` 时直接停止。
6. 主循环开头保留 `Enter`。
7. 制造商定位段替换为：

```text
Backspace
打开制造商页面后：鼠标移到屏幕中心，向下滚动 `10` 格，整屏 OCR 找 `斯巴鲁` 并点击。
```

8. 进入斯巴鲁列表后，后续保持原定流程：
   - 找 `IMPREZA 22B-STI`
   - 使用已保存的 `GridGeometry`
   - 每轮更新 `CellOccupancy`
   - 找有效 `全新`
   - 用方向键移动到目标格
   - 执行固定按键序列
   - 固定按键序列完整执行后扣 `32` 技术点
   - 记录本轮命中的滚动位置，下一轮直接滚动到附近继续找

## 识别策略

- 制造商 `斯巴鲁` 使用向下滚动 `10` 格 + OCR 点击，不再录制或回放键盘路径。
- `IMPREZA 22B-STI` 使用 PaddleOCR PP-OCRv5。
- `全新` 使用 PaddleOCR PP-OCRv5。
- `IMPREZA 22B-STI` 和 `全新` 都先精确匹配，失败时做小范围模糊匹配。
- 当前脚本只使用 OCR 文本识别。
- 普通模式截图在内存中转 PNG base64 发给常驻 PaddleOCR 进程，不保留截图。
- 调试模式会保存原始截图、实际送入 OCR 的缩放图、OCR 原始返回和元信息到 `debug/screenshots/`，并在下一次调试模式启动时清空旧截图。
- OCR 阶段失败会保存识别文本和坐标到 `debug/ocr-last-*.txt`。

## 叠加层

叠加层用于显示脚本正在工作：

- 当前状态机步骤。
- 虚拟格子。
- 当前 `CellOccupancy`。
- 有效目标。
- 最终目标格。
- OCR 已识别到的符合条件字段会用细框显示，主要包括 `IMPREZA 22B-STI` 和 `全新`。
- 耗时和失败次数。
- 运行模式、下一步操作、轮数、调试步数。
- 运行期虚拟车辆列表摘要。

截图前必须隐藏叠加层，避免污染识别。

## 运行期虚拟列表

- 当前脚本决策只使用本次进程内的虚拟列表，启动时不读取旧文件。
- 每次运行会实时覆盖保存 `state/virtual-vehicle-list.json`，供后续独立脚本读取。
- 车辆列表每向下滚动一格，虚拟列表的全局列坐标加一。
- OCR 当前屏幕后，把当前可见行列映射到全局列坐标。
- 虚拟列表状态：
  - `0`：非斯巴鲁或未确认制造商。
  - `1`：斯巴鲁，但不是目标车。
  - `2`：目标车辆 `IMPREZA 22B-STI`，但当前没有待处理的 `全新`。
  - `3`：目标车辆且有 `全新`，代表还有待点技术点的车辆。
  - `4`：可删车辆。
  - `5`：目标车且有 `900`，用于全自动流程选择开蓝图车辆。
- 命中有效目标后记录当前滚动偏移量；如果虚拟列表里还有 `3`，下一轮进入列表后优先滚到能看到这个 `3` 的位置，并直接用方向键处理，不做 OCR。
- 固定操作完成后，刚处理的格子从 `3` 变成 `4`，进入后续自动删车阶段。
- 自动停止规则：
  - 自动点技术点必须没有状态 `3`，并确认目标车型区间末尾的下一个格子是 `0` 或 `1`；
  - 自动删车必须没有状态 `4`，并确认目标车型区间末尾的下一个格子是 `0` 或 `1`；
  - 如果一直没找到状态 `2/3/4/5`，只有连续观察到列表末尾出现状态 `0` 才能停止；状态 `1` 仍然表示斯巴鲁车辆，不能作为“滚出斯巴鲁制造商区域”的证据；
  - 则认为目标车的待点技术点已处理完。本轮固定操作完成后停止脚本。
  - 如果下一个格子不存在，不停止，因为后续新列可能还有新车。
- 只有虚拟列表中没有 `3` 时，才按当前可见列数跳过已知非 `3` 区间：
  - 如果已知非 `3` 列数量大于可见列数，先跳到最后一列已知非 `3` 刚好可见的位置，不 OCR。
  - 如果跳完后会露出新列，才进行 OCR。
- 调试模式下写入 `debug/virtual-list-edits-*.log`，记录所有虚拟列表编辑。
