# 自动删车虚拟列表重构文档

## 范围

本轮只重构 `FH6VehicleDeleteOcr.exe` 的车辆列表查找可删车流程。

暂不重构：

- 自动点技术点流程。
- 全自动总控找 `900` 分开蓝图车辆流程。
- 删车固定按键序列。
- OCR 字段识别规则。

## 当前问题

现有 `FindDeletableVehicleCell()` 把多个职责串在一个循环里：

- 当前已知状态 `2` 是否能直接处理。
- 后面是否有已知状态 `2` 可以跳转。
- 是否应该 OCR。
- 是否应该激进跳过。
- 是否已经到斯巴鲁列表边界。
- 是否完成删车。

这些判断不是一个统一计划，而是按代码顺序叠加。调试时容易出现“明明还有已知可删格却先 OCR”“没有已知可删格时滚动策略混乱”“删完后选中位置和虚拟表不同步”等问题。

## 稳定对象

- `VirtualVehicleList`：维护状态 `2/3/4/5` 和滚动 offset。
- `BuildVehicleGridObservation()` / `ApplyVehicleGridObservation()`：只做 OCR 观察和写表。
- `DeletePlanner`：删车阶段唯一的下一步决策者。
- 删车执行器：沿用 `MoveDeleteSelectionToCell()`、`RunDeleteVehicleSequence()`、`MarkVehicleCellDeleted()`。

## 动作集合

```text
Select(localCell)
Scroll(ticks, reason)
Observe(reason)
Stop(reason, message)
```

含义：

- `Select`：当前可见范围已有可删车辆，方向键移动到该格并执行删车序列。
- `Scroll`：虚拟表已经能判断应该滚到更有价值的位置。
- `Observe`：当前页还缺可信状态，需要 OCR 当前车辆列表区域并写表。
- `Stop`：确认没有剩余可删车辆，结束当前子程序。

## 可删定义

删车目标是：

```text
状态 4：可删车辆
```

状态 `2` 只是目标车，不直接删除；同格状态 `3` 不删；旧版状态 `5` 当前不再写入，也不属于删车目标。

## 决策顺序

每轮先读虚拟表，再决定动作：

```text
if 当前内部选中格仍是可删车辆:
    Select(当前选中格)

else if 当前可见范围有可删车辆:
    Select(最左最上的可删车辆)

else if 当前可见范围没有可删车辆，但后方已知有可删车辆:
    Scroll(滚到能看到该可删车辆)

else if 上一次 OCR 已确认没有可处理斯巴鲁:
    Stop(斯巴鲁列表结束)

else if 当前可见范围已知出现状态 0:
    Stop(到达非斯巴鲁或未知制造商区域)

else if 虚拟表满足删车完成边界:
    Stop(没有状态 4，目标车型区间尾部已确认是 0 或 1)

else if 从列表开头到当前观察范围一直没有状态 2/3/4/5，且连续观察到的末尾已经出现状态 0:
    Stop(斯巴鲁制造商区域内没有目标车型)

else if 当前可见范围还有未知格子:
    Observe(当前页未完整观察)

else if 当前没有可删车辆，且从当前 offset 开始有连续已知不可删列:
    Scroll(连续已知不可删列数量 - 最左侧保留的 1 列已知格子)

else:
    Scroll(1)
```

重要约束：

- 当前可见已有可删车辆时，绝不能先 OCR 或滚动。
- 当前内部选中格仍可删时，直接继续删，不重新移动。
- OCR 只负责填表，不直接决定是否删车或滚动。
- 删除完成后，游戏选中框回到当前页面第一格，脚本也必须同步重置内部选中格。
- 删除后必须调用虚拟表前移逻辑，不能只把当前格标空。

## 检查点

- 衔接启动读取旧表后，如果可见范围已有状态 `2`，第一步必须直接删。
- 删除一辆车后，如果新的当前第一页可见范围仍有可删车，下一轮不能先 OCR。
- 如果没有已知可删车，但当前页没有完整观察，必须先 OCR。
- 如果当前页完整观察且没有可删车，才允许跳过已知不可删区间。
- 当前页开头识别到状态 `0` 或 `1` 不能直接停止。必须继续 OCR/滚动，直到已经观察到目标车型区间，并确认目标车型区间末尾的下一个格子是 `0` 或 `1`；自动删车还要确认边界前没有状态 `4` 可删车辆后，才停止当前子程序。状态 `2` 只是目标车，不再直接当作删车目标。
- 如果一直没有观察到目标车型状态 `2/3/4/5`，只有连续观察到的列表末尾出现状态 `0` 才能停止；状态 `1` 表示仍然是斯巴鲁车辆，不能作为“没有目标车”的停止边界。
