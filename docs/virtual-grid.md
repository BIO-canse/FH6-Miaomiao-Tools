# 车辆列表辅助定位方案

当前主链路使用 OCR/文本定位车辆名和 `全新`，虚拟格子只负责把 OCR 坐标映射到车辆格子。

## 用途

车辆列表里目标是定位 `IMPREZA 22B-STI`，然后进入该车的操作菜单和技术点页面。

这个方案只负责在车辆列表中定位目标车辆。

## 两层模型

1. `GridGeometry`：车辆列表的固定像素几何。
2. `CellOccupancy`：当前屏幕内每个格子是否为目标车辆。

`GridGeometry` 在同一台电脑、同一分辨率、同一缩放设置下固定。滚动和车辆列表内容变化只影响 `CellOccupancy`。

## GridGeometry

几何不再通过 OCR 车名推断。

首次没有 `config/user-settings.json` 时，程序会询问：

```text
visible_rows
visible_columns
```

然后弹出框选层，让玩家鼠标框选“最左上角完整车辆格子”。保存字段：

```text
grid_cell_left
grid_cell_top
grid_cell_width
grid_cell_height
```

后续直接构建格子：

```text
cellLeft = grid_cell_left + col * grid_cell_width
cellTop  = grid_cell_top  + row * grid_cell_height
hitRect[row][col] = cellLeft/cellTop/grid_cell_width/grid_cell_height
```

保存后几何立即锁定，不再滚动暴露三列车辆，不再根据 `IMPREZA 22B-STI` 字段估算格子大小。

## CellOccupancy

字段建议：

```text
isTarget[row][col]: bool
lastSeenScore[row][col]: float
lastSeenAt[row][col]: timestamp
```

每次滚动后：

1. OCR 找当前屏幕内所有 `IMPREZA 22B-STI`。
2. 用识别框中心点命中 `hitRect[row][col]`。
3. 映射成功的格子置为 `true`。
4. 没有映射到的格子保持 `false`。

`false` 表示当前不把该格子当目标车处理。这样最多漏处理，不会误处理。

## 运行期虚拟列表保存

`VirtualVehicleList` 会随着滚动持续扩展，并实时覆盖保存到 `state/virtual-vehicle-list.json`。

独立启动时不会读取这个文件，避免使用上次运行后的过期车辆顺序。全自动流程中，自动点技能点每轮开始也不会读取旧表内部车辆状态，会重新 OCR 扫表并覆盖该文件；只有同一轮后续的删车衔接和删车后的找状态 `5` 会读取这个文件，用本轮已经建立并更新过的车辆状态提速。

虚拟列表规则：

- 第一个格子固定当作占位车，永远不会被点技能点或删除。
- OCR 写入虚拟列表时忽略所有已经存在于虚拟表里的已知格子，避免反复写已经确认过的车辆状态。
- 用户修正原话：“跟第一列没关系，总之就是OCR自动写表不会反复写已知格子。”
- 自动点技术点、自动删车和全自动找状态 `5` 每次 OCR 车辆列表时都会走统一封装，一次性识别 `IMPREZA 22B-STI`、`全新`、`斯巴鲁`、`600`、`900`，然后统一写入虚拟列表。
- 只有格子内识别到 `斯巴鲁` 的非目标车才记为状态 `1`。没有 `斯巴鲁` 的格子记为状态 `0`，代表其它制造商或未确认制造商，不再参与状态 `1/2/3/5` 的连续区间跳过。
- 如果当前列表区域没有可处理的 `斯巴鲁`，或者只识别到全局列表第一个占位格是 `斯巴鲁`，脚本认为已经滚出斯巴鲁制造商区域或没有剩余车辆。
- 列表边界不能用“当前页第一个 `0` 或 `1`”判断。开头是 `0` 或 `1` 不能直接退；只有已经观察到目标车型区间后，目标车型区间末尾的下一个格子确认是 `0` 或 `1`，才算完成边界。自动点技术点以 `2/3/4/5` 作为目标车型区间证据，自动删车同样要先确认 `2/4` 这类可删目标已不存在，再用区间末尾后的 `0/1` 停止。
- 独立启动或缺少可信状态时，首轮先 OCR 当前屏幕；衔接启动时优先使用上一段脚本保存的虚拟列表状态，已有状态 `2` / `5` 可直接跳转或处理。
- 对找状态 `5` 来说，只有经过统一 OCR 确认没有 `900` 的目标车格，才会被当作“已知非 5”参与跳过。单纯从点技能或删车继承来的状态 `2` 不能证明不是状态 `5`。
- 激进滚动的目标是显示尽可能多的未知列，同时最左侧保留 1 列已知格子。
- 如果连续已知区长度为 `run`，激进滚动滚动 `run - 1` 列；滚动后未知列从第 2 列开始进入 OCR 范围。

状态码：

```text
0 = 非斯巴鲁或未确认制造商
1 = 斯巴鲁，但不是目标车
2 = 目标车，但没有有效“全新”；自动删车阶段会删除该状态
3 = 目标车，并且有有效“全新”等待处理
4 = 旧版兼容状态，目标车且曾识别到 600；删车阶段也会按可删处理
5 = 目标车且有 900，用于全自动流程选择开蓝图车辆
```

## 选择目标车辆

如果屏幕上有多个有效目标车辆格子，选择最左最上的一个：

```text
rank = col * 1000 + row
```

方向键移动：

```text
Right x targetCol
Down  x targetRow
Enter
```

`targetRow` 必须在 `0` 到 `visible_rows - 1` 之间。

## 叠加层

如果启用车辆列表辅助定位，叠加层显示：

- 蓝色边框：车辆列表格子。
- 无填充：空格子、状态 `1`、或 `全新` 但不属于目标车。
- 半透明淡黄色填充：状态 `2`，目标车但没有有效 `全新`。
- 半透明淡红色填充：状态 `3`，目标车且有有效 `全新`。
- 细蓝色边框：车辆列表格子。
- 青绿色细框：OCR 识别到的车名字段。
- 橙色细框：OCR 识别到的 `全新` 字段。
- 紫色细框：OCR 识别到的 `斯巴鲁` 字段。
- 每个格子显示 `G全局列,R行 S状态码`，滚动后 `G` 会跟随 `CurrentOffset` 更新，未知格子显示 `S?`。
- 左上角状态：当前状态、下一步、轮数、OCR 摘要、失败次数。

截图前必须隐藏叠加层，避免污染 OCR。
