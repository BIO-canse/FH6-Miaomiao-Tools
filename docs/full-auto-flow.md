# 全自动总控流程

## 用户前置准备

全自动流程面向挂机获取超级抽奖：自动刷技术点、自动点技术点、自动删除多余车辆、自动补买车辆，然后进入下一轮。

开始前必须准备：

1. 车库里有一辆改装后的 `1998 斯巴鲁 IMPREZA 22B-STI VERSION`，性能等级 `S2`，性能分 `900`。建议下载速度快的调教，保证能在规定时间内完成蓝图赛事；这辆改装车放在车库里。
2. 另有一辆原厂 `1998 斯巴鲁 IMPREZA 22B-STI VERSION`。开始运行全自动脚本前，当前驾驶车辆需要是这辆原厂车。
3. 难度设置开启自动转向，变速箱使用自动挡。
4. AI 难度最好选择“所向披靡”，避免赛事开始前弹出是否增加难度的询问。
5. 蓝图代码 `175 882 167` 必须已经加入“我的收藏”，并且处于“我的收藏”的第一位。程序进入收藏蓝图列表后会直接选择第一项，参考 `assets/favorite-blueprint-first.png`。
6. 最终按 `Enter` 确认开始前，确认当前驾驶车辆是原厂车，并且输入法已经切到英文，避免中文输入法卡住按键。
7. 确认后程序等待 `10` 秒，等待结束时必须处于“大世界”可操作状态，参考 `assets/overworld-start.png`。

首次运行或重设设置时：

1. 进入“我的车辆”选车页面，参考 `assets/vehicle-grid-page.png`。
2. 填写完整可见车辆列数和行数，只统计完整格子；左侧车辆详情栏、最左/最右边缘露出的半个格子都不要算。
3. 用鼠标画框，精确框住右侧所有完整可见车辆格子的整体区域。不要框左侧车辆详情栏、顶部制造商标签栏、底部按键提示，也不要框边缘不完整的半格。
4. 输入当前已有技术点数量。
5. 最后按 `Enter` 确认开始，程序等待 `10` 秒后真正执行。

分辨率和性能：

- 本工具支持任意分辨率，首次设置会按当前画面框选并保存完整车辆格子区域。
- 如果玩家更改分辨率、窗口缩放或 UI 比例，需要选择 `3` 重设设置重新框选。
- 建议把画质调到最低或较低，减少菜单动画和加载耗时，避免性能差的电脑超过脚本预留等待时间。
- 玩家需要预留足够的 CR。全自动流程会自动补买目标车，如果 CR 不足，买车确认流程可能卡住。
- 脚本会按首次框选和 OCR 动态识别车辆列表，车库里有其它车辆或干扰车辆不会影响流程；只需要保证车辆、难度、收藏蓝图和起始状态这些前提条件满足。

## 总控目标

`FH6FullAuto.exe` 负责把已有脚本串成一个完整闭环：

1. 从大世界进入车库标准位。
2. 自动点车辆技能点。
3. 自动删除不再需要的车辆。
4. 找到用来开蓝图的目标车。
5. 进入收藏蓝图并运行 1 分钟刷技术点循环。
6. 刷技术点循环每完成 1 轮，内部技术点计数加 `10`；到 `999` 后安全退出刷技术点循环。
7. 回到大世界进入车库标准位，再次点技能点。
8. 无限循环。

总控急停：

- `Space+C`：立即急停。正在运行子程序时会直接结束子程序。
- `Space+V`：总控安全急停。
  - 如果没有子程序在运行，主程序直接退出。
  - 如果子程序正在运行，总控只给该子程序写安全退出请求文件。
  - 子程序跑完当前轮并退出后，总控再退出。
  - 总控因为技术点计数到 `999` 而让 `MinuteWLoop.exe` 安全退出，不等于用户按 `Space+V`，不会让总控退出。

阶段间隔：

- 总控在每个子程序启动前预留 `1` 秒。
- 总控在每个子程序正常退出后预留 `1` 秒。
- 总控在连续固定流程之间预留 `1` 秒，例如 `E. 刷技术点结束后的返回流程` 结束后，再执行 `A. 大世界自动进入车库标准位`。
- 这个间隔用于避免上一个流程刚结束、游戏焦点或菜单动画还没稳定时，下一个子程序衔接启动直接丢按键。

## 启动前置人工介入

总控启动后必须先集中处理所有会打断流程的人工介入，不只包括控制台询问，也包括鼠标框选、需要用户按键确认的设置动作。后续子程序必须使用命令行参数或已有配置自动继续，不能在全自动流程中途突然停下来等用户操作。

当前会遇到的人工介入：

| 来源 | 询问 | 总控处理方式 |
| --- | --- | --- |
| 总控自身 | 运行模式：正常 / 调试 / 重设设置 | 保留在总控最开始。后续子程序全部传 `--mode normal`，不再询问。 |
| `UserSettings` | 首次运行或重设时输入可见完整车辆行列数 | 总控最开始处理并保存到 `config/user-settings.json`。后续子程序读取，不再询问。 |
| `UserSettings` | 首次运行或重设时鼠标框选完整可见车辆格子整体区域 | 总控最开始处理并保存到 `config/user-settings.json`。这是“拉表格”类动作，必须前置。后续子程序读取，不再要求用户框选。 |
| 技能点脚本 | 当前技术点总数 | 总控启动阶段询问一次，然后写入运行期技术点计数文件；总控每次都用 `--handoff --skill-points <当前计数>` 衔接调用技能点脚本。 |
| 技能点衔接启动 | 当前技术点总数 | 不询问，不显示启动确认，不等待 10 秒；衔接模式读取总控维护的运行期技术点计数。没有总控传值时才默认 `999`。 |
| 制造商定位 | 进入斯巴鲁制造商 | 不再录制或回放方向键路径。打开制造商页后，统一复刻买车前置的方式：鼠标移到屏幕中心，滚轮向下滚动到底，整屏 OCR 找 `斯巴鲁` 并点击。车库制造商页和买车前置制造商页使用不同的 UI 坐标缓存标签，不能共用缓存。 |
| 删除车辆脚本 | 启动模式 | 总控只使用 `--handoff` 衔接启动，不询问，不显示启动确认，不等待 10 秒。 |
| 1 分钟刷技术点脚本 | 退出方式 | 总控传 `--handoff --skill-points-state-file`，衔接启动时跳过脚本自己的开局 10 秒等待；脚本每轮加 `10`，到 `999` 后自动跑完当前轮退出；`--safe-stop-file` 只用于用户安全退出。 |

前置原则：

```text
凡是会让用户输入、确认、框选、切换选择、或根据屏幕状态手动配合的动作，都必须放在总控正式循环开始之前。
正式循环开始后，只允许自动 OCR、自动键鼠、自动读写配置、自动启动子程序。
```

## 第一次实际执行顺序

这一节按 `FH6FullAuto.exe` 当前真实调用顺序写。不要把后面的 `A/B/C/D/E` 直接串起来理解；那些只是固定动作片段，真实顺序以本节和“后续真实循环流程”为准。

第一次启动到进入循环前：

1. 总控启动，先处理所有前置人工介入：
   - 选择运行模式。
   - 读取或重设格子行列、完整可见车辆格子区域。
   - 询问本次当前技术点总数。
2. 所有准备工作完成后，总控提示最终启动。
3. 玩家按 `Enter`，非调试模式等待 `10` 秒；调试模式进入单步流程。
4. 执行固定流程 `A. 大世界自动进入车库标准位`。
5. 等待 `1` 秒阶段间隔。
6. 衔接调用自动点技能点子脚本：

```text
FH6SkillPointOcr.exe --task skill --handoff --skill-points <启动时输入的技术点> --skill-points-state-file <计数文件>
```

这个第一次技能点子脚本也是衔接模式：不会再提示按 `Enter`，不会再等待 `10` 秒。它会自己进入斯巴鲁车辆列表、找待点技术点的目标车、执行点技能点流程，并把剩余技术点写回计数文件。

注意：衔接模式只跳过子程序启动确认，不跳过子程序内部进入制造商的固定动作。自动点技能点和衔接删车进入斯巴鲁列表时仍然必须先执行 `Enter -> 等待 1 秒 -> Backspace -> 等待页面稳定 -> 滚动到底 -> OCR 点击斯巴鲁 -> 等待页面进入 -> 鼠标停到第二行最后一列右侧屏幕边缘`。

第一次自动点技能点子脚本退出后，等待 `1` 秒阶段间隔，才进入下面的“后续真实循环流程”。也就是说第一次实际执行顺序不是 `A -> B -> C -> D -> E`，而是：

```text
启动前置
最终确认启动，等待 10 秒
A. 大世界自动进入车库标准位
等待 1 秒阶段间隔
衔接调用自动点技能点子脚本
等待 1 秒阶段间隔
进入后续循环
```

## 后续真实循环流程

从第一次普通自动点技能点子脚本退出后，主程序进入无限循环。每一轮真实顺序如下：

1. 衔接调用自动删车子脚本：

```text
FH6VehicleDeleteOcr.exe --task delete --handoff
```

删车子脚本是衔接模式：会进入斯巴鲁车辆列表，完整读取上一段自动点技能点留下的虚拟列表状态，优先直接处理已知状态 `2`。每次 OCR 都会一次性识别 `IMPREZA 22B-STI`、`全新`、`斯巴鲁`、`600`、`900` 并统一写表；删除状态 `2` 后，虚拟列表同步前移并保存，直到确认没有剩余可删车辆后退出。

2. 子程序退出后等待 `1` 秒阶段间隔。
3. 主程序按 `Esc`，等待 `0.5` 秒，从删车后的车辆列表退出到上一级。
4. 找状态 `5` 流程先执行 `Enter -> 等待 0.5 秒 -> Backspace -> 等待 0.5 秒 -> 滚动到底 -> OCR 点击斯巴鲁 -> 等待页面进入 -> 鼠标停到第二行最后一列右侧屏幕边缘`，重新进入斯巴鲁列表起点。
5. 开始查找状态 `5`。状态 `5` 查找规则：
    - 总控先重新读取删车子脚本保存后的虚拟列表。
    - 如果虚拟列表当前可见范围已有状态 `5`，直接使用，不 OCR。
    - 如果虚拟列表已知状态 `5` 不在当前可见范围内，直接跳转到能看到该格的位置。
    - 否则 OCR 当前可见格子里的 `IMPREZA 22B-STI`。
    - OCR 当前可见格子里的 `900`。
    - 两者映射到同一个虚拟格子时，把该格记为状态 `5`。
    - 如果当前可见范围没有状态 `5`，先确认仍在斯巴鲁区域，再按虚拟列表做激进滚动继续找；只有经过统一 OCR 确认没有 `900` 的目标车列，才会被当作“已知非 5”跳过。
    - 如果当前 OCR 已经在可处理格子内发现状态 `0`，说明 `0` 后面不可能再有状态 `5`；但不是可见 `0` 立刻退出，而是先把当前可见范围内第一个 `0` 前面的目标段处理/确认完。
    - 如果第一个 `0` 前面已扫描/已知范围没有状态 `5`，说明后面不可能再出现状态 `5`，直接默认第一页第一列第一行就是可用车辆。
    - 如果是在列表起点看到状态 `0`，直接默认第一列第一行就是可用车辆；如果是滚动后看到状态 `0`，先执行 `Esc -> 0.5 秒 -> Enter -> 0.5 秒 -> Backspace -> 0.5 秒 -> 滚动到底 -> OCR 点击斯巴鲁 -> 等待页面进入 -> 鼠标停到第二行最后一列右侧屏幕边缘` 回到列表起点，再默认第一列第一行就是可用车辆。
6. 找到状态 `5` 后，用方向键移动到对应车辆格；没找到状态 `5` 时不额外移动，使用第一页第一列第一行。
7. 执行固定流程 `B. 选中状态 5 后进入后续菜单`。
8. 执行固定流程 `C. 买车前置流程`。
9. 在 `C` 中间调用自动买车子脚本。买车不是固定 31 辆，而是把车库里指定车型的状态 `3` 补充到 31 辆：

```text
buyRounds = max(0, 31 - 当前虚拟表状态3数量)
```

```text
SpaceDownEnterLoop.exe --rounds <buyRounds> --startup-delay-ms 0 --safe-stop-file <文件>
```

10. 买车脚本退出后，等待 `1` 秒阶段间隔，主程序继续完成 `C` 末尾的 `Esc x4` 返回动作。
11. 执行固定流程 `D. 进入创意中心收藏蓝图`。
12. 调用 1 分钟刷技术点子脚本：

```text
MinuteWLoop.exe --handoff --safe-stop-file <文件> --skill-points-state-file <计数文件>
```

`MinuteWLoop.exe` 衔接启动时跳过自己的开局 10 秒等待；每完成一轮把技术点计数加 `10`，到 `999` 后安全退出。

13. `MinuteWLoop.exe` 退出后等待 `1` 秒阶段间隔。
14. 执行固定流程 `E. 刷技术点结束后的返回流程`：
    - 执行 `Down x4, Enter, 等待 1 秒, Enter, 等待 20 秒`。
15. 等待 `1` 秒阶段间隔后，执行 `A. 大世界自动进入车库标准位`。
16. 等待 `1` 秒阶段间隔后，衔接调用自动点技能点子脚本：

```text
FH6SkillPointOcr.exe --task skill --handoff --skill-points <当前计数> --skill-points-state-file <计数文件>
```

衔接自动点技能点使用总控维护的当前技术点计数，不再询问玩家当前技术点；它不会读取上一轮虚拟表内部车辆状态，而是重新进入斯巴鲁车辆列表，重新 OCR 扫表并覆盖 `state/virtual-vehicle-list.json`，点技能点直到本轮结束或技术点不足。

17. 衔接自动点技能点退出后，等待 `1` 秒阶段间隔，本轮结束，下一轮从第 1 步“衔接调用自动删车子脚本”重新开始。

注意：虚拟表只在同一轮内部复用，链路是“本轮自动点技能点建表 -> 本轮删车读取并更新 -> 本轮找状态 5 再读取”。进入下一轮自动点技能点时，旧表内部车辆状态作废，必须重新扫。

简写如下：

```text
衔接删车
等待 1 秒阶段间隔
找状态 5
B. 进入后续菜单
C. 买车前置 + 调用买车脚本 + 返回
D. 进入创意中心收藏蓝图
调用 MinuteWLoop 补技术点到 999
等待 1 秒阶段间隔
E. 返回流程
等待 1 秒阶段间隔
A. 大世界自动进入车库标准位
等待 1 秒阶段间隔
衔接自动点技能点
等待 1 秒阶段间隔
回到本循环开头
```

## 找状态 5 开蓝图车辆

状态 `5` 表示：

```text
目标车 IMPREZA 22B-STI
并且同一车辆格子里 OCR 到 900
```

浮层显示状态 `5` 时，会在对应车辆格子的蓝色边框内部画一个小圆。

查找流程：

1. 主程序先从删车后的车辆列表按 `Esc`，等待 `0.5` 秒。
2. 执行 `Enter -> 等待 0.5 秒 -> Backspace -> 等待 0.5 秒 -> 滚动到底 -> OCR 点击斯巴鲁 -> 等待页面进入 -> 鼠标停到第二行最后一列右侧屏幕边缘`，重新进入斯巴鲁列表起点。
3. 优先读取删车脚本保存后的虚拟列表；可见范围已有状态 `5` 就直接使用，已知状态 `5` 在后方就直接跳转过去。
4. 如果没有可直接使用的状态 `5`，OCR 当前可见格子。一次 OCR 统一识别 `IMPREZA 22B-STI`、`全新`、`斯巴鲁`、`600`、`900` 并写入虚拟列表。
5. `IMPREZA 22B-STI` 和 `900` 映射到同一个虚拟格子时，把该格记为状态 `5`。
6. 同时 OCR `斯巴鲁`；如果当前页已经出现状态 `0`，说明 `0` 后面不可能再有目标段，但要先处理/确认第一个 `0` 前面的当前可见目标段。
7. 状态 `5` 也是目标车型区间里的斯巴鲁车，所以如果第一个 `0` 前面没有状态 `5`，后面就不可能再出现状态 `5`。此时默认第一页第一列第一行就是可用车辆。
8. 如果是在列表起点看到状态 `0`，默认第一页第一列第一行就是可用车辆；如果是滚动后看到状态 `0`，执行 `Esc -> 0.5 秒 -> Enter -> 0.5 秒 -> Backspace -> 0.5 秒 -> 滚动到底 -> OCR 点击斯巴鲁 -> 等待页面进入 -> 鼠标停到第二行最后一列右侧屏幕边缘` 复位后再默认第一格。
9. 如果当前可见范围没有状态 `5` 且还没到状态 `0` 边界，按虚拟列表做激进滚动，跳过已知非 `5` 区间。只有经过统一 OCR 确认没有 `900` 的目标车格，才算已知非 `5`。
10. 找到状态 `5` 后，用方向键移动到该格。
11. 如果目标格已经选中，则不额外移动。

选中状态 `5` 后，主程序依次执行固定流程 `B`、`C`、`D`，然后启动 `MinuteWLoop.exe`。

## 刷技术点循环

总控启动：

```text
MinuteWLoop.exe --handoff --safe-stop-file <文件> --skill-points-state-file <计数文件>
```

`MinuteWLoop.exe` 衔接启动时跳过自己的开局 10 秒等待，之后执行 1 分钟循环：

```text
确保 W 松开
Enter
Enter 后 1 秒，按住 W
等待 1 分钟
松开 W
X
等待 1 秒
确保 W 松开
Enter
等待 10 秒
```

W 不会在 Enter 前按下，避免菜单选项被 W 移动。

技术点计数规则：

1. 总控维护一个运行期技术点计数文件。
2. 自动点技能点脚本每点完一辆车，就把计数扣 `32` 并写回文件。
3. `MinuteWLoop.exe` 每完成 1 轮，就把计数加 `10` 并写回文件。
4. 技术点计数最高封顶 `999`。
5. `MinuteWLoop.exe` 发现计数到 `999` 后，不再进入下一轮，释放 W 并安全退出。
6. 总控读取计数文件，后续自动点技能点脚本继续使用该值。

返回流程见“主程序固定操作流程 / E. 刷技术点结束后的返回流程”。

## 关键实现约束

- 子程序启动时总控浮层必须隐藏，避免污染子程序 OCR 截图。
- 子程序退出后再显示总控浮层。
- 子程序启动前、正常退出后，以及连续固定流程之间，必须保留 `1` 秒阶段间隔。
- 固定 UI OCR 使用当前显示器整屏截图；车辆格子 OCR 才使用车辆格子区域裁剪。任意 OCR 失败时都会额外保存 `debug/failure-fullscreen-*.png` 整屏截图用于排查。
- 浮层必须同时显示“大阶段”和“下一步”。大阶段用于定位当前属于 `A. 大世界自动进入车库标准位`、`子程序: 自动点技能点`、`找状态 5 开蓝图车辆`、`C. 买车前置流程`、`D. 进入创意中心收藏蓝图`、`E. 刷技术点结束后的返回流程` 或阶段间隔。
- 涉及菜单跳转后马上 OCR 的固定 UI 阶段，在截图前额外等待 `1` 秒，避免界面动画或加载导致截早。
- 自动点技术点、自动删车、车辆列表内查找状态 5 等“我的车辆”页面内部操作期间，鼠标停在当前可见第一个车辆格子中心；制造商页、创意中心、蓝图菜单等其它阶段，鼠标空闲时停到屏幕最右下角。
- 总控会缓存本次进程里 OCR 找到的固定 UI 点击坐标；缓存只在 `FH6FullAuto.exe` 运行期间存在，不写入文件，主程序退出后消失。
- UI 坐标缓存只用于非车辆内容的固定 UI：制造商页的 `斯巴鲁`、`创意中心`、`最新最热`、`我的收藏`。第一次 OCR 找到后记录中心点；后续循环命中缓存时，因为跳过了 OCR 天然等待，会先等待 `0.5` 秒再点击缓存坐标。
- 车辆相关内容不使用 UI 坐标缓存，包括 `IMPREZA 22B-STI`、`全新`、`900`、状态 `2`、状态 `3`、状态 `5`、技术点固定序列和删车固定序列。
- 总控主动安全退出子程序和用户按 `Space+V` 必须区分。
- `Space+V` 是统一安全退出快捷键；全自动总控在子程序运行期间会把它转换为对子程序的安全退出请求，并在子程序退出后再结束主程序。
- 子程序由总控传 `--safe-stop-file`，通过文件请求安全结束。
- `MinuteWLoop.exe` 的安全退出必须跑完当前 1 分钟循环，不在等待中直接断。
- 技术点计数是总控的稳定状态，必须通过运行期计数文件在子程序之间传递；不能让后续衔接技能点脚本盲目使用固定 `999`。
